# 4.1.3 后端实现

本项目后端主要的技术栈包括：选用基于 Python 的 FastAPI 作为后端框架，提供高性能的异步 REST API 服务；使用 Redis 作为缓存层，存储临时会话数据；选用 SQLite 进行轻量级的数据持久化；使用 FFmpeg 进行视频处理。

后端主要提供 REST 风格的接口响应给前端应用，包括 Web 端和小程序端。

本项目的后端职责有以下几点：

**负责模型推理任务的转发。** 视频上传、手语翻译模型推理都是耗时任务，对于这部分专门设计了异步处理流程。前端上传视频后，后端立即返回任务 ID，模型推理在后台完成，前端通过轮询或 WebSocket 获取结果。

**负责数据的持久化。** 提供对用户翻译历史、收藏语句、个性化配置等内容的更改与持久化。对于大文件的持久化，如用户上传的视频文件，保存在本地文件系统并按日期归档，定期清理过期文件。在实现上，后端接口采用流式处理，避免大文件上传占据过多内存。

**负责用户的鉴权验证。** 本部分基于 JWT（JSON Web Token）实现，为每个用户分配唯一的 token，用于识别用户身份和保护隐私数据。

# 4.1.4 模型端实现

本项目的核心手语翻译模型在 AutoDL 云平台上进行推理部署，模型实现主要基于 PyTorch 深度学习框架。

**手语翻译模型**采用 Uni-Sign（ICLR 2025）架构，该模型基于 ST-GCN 时空图卷积网络，能够同时处理姿态关键点序列和 RGB 视频帧，实现高精度的连续手语翻译。使用的检查点基于 CSL Daily 数据集训练，支持 1000+ 个手语词汇的识别。

**心理咨询模型**采用 EmoLLM（基于 Qwen2-7B 微调），专门为聋哑人群体提供心理健康支持。模型集成了 RAG（检索增强生成）技术，融合心理学知识库，能够提供更专业的心理建议。

**语音合成模型**采用 ChatTTS，提供高质量的中文语音合成服务。由于 ChatTTS 依赖的 PyTorch 版本与主环境冲突，本项目创新性地采用独立虚拟环境部署，通过内部端口转发实现服务隔离，确保系统稳定性。

# 4.1.5 系统部署

AI 应用的部署一大难点在于 GPU 服务器的获取与模型服务的对外暴露。由于模型推理需要 GPU 算力，团队选用 AutoDL 云平台进行部署。

然而，AutoDL 平台默认仅提供 Jupyter Notebook 容器环境，对外不开放端口与公网 IP。为解决这一问题，团队采用以下方案：

**端口映射与反向代理。** 利用 AutoDL 提供的公网 IP 和端口映射功能，将内部服务端口（6006、6009）映射到外部可访问的地址。主 API 服务监听 6006 端口，TTS 独立服务监听 6009 端口，外部统一通过 6006 访问。

**TTS 服务隔离部署。** 为解决 PyTorch 版本冲突（主环境需要 2.8+，ChatTTS 需要 2.2），创建独立的 Python 虚拟环境（tts_env），单独部署 TTS 服务。主 API 通过 HTTP 转发将 TTS 请求转发至内部 6009 端口，对外呈现统一接口。

**微服务架构。** 项目整体采用模型推理微服务 + 前端分离部署架构：
- 模型端：基于 FastAPI 提供手语翻译、心理咨询、语音合成接口
- 前端：Vue.js 开发的 Web 应用，可打包为小程序或 H5
- 通信：前端通过 HTTPS 调用后端 API，JSON 格式交互

当前系统已在 AutoDL 平台稳定运行，支持并发访问和长时间推理任务。
